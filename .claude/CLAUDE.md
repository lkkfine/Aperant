## Build System

All build artifacts output to the project-root `build/` directory:

```
build/
‚îú‚îÄ‚îÄ main/       # Electron main process (index.js)
‚îú‚îÄ‚îÄ preload/    # Preload scripts (index.mjs)
‚îú‚îÄ‚îÄ renderer/   # React UI (index.html + assets/)
```

Key commands:
- `npm run build` ‚Äî compile to `build/`
- `npm run build:start` ‚Äî compile + launch Electron from `build/`
- `npm run build:clean` ‚Äî remove `build/`
- `npm run dev` ‚Äî dev mode with HMR (also outputs to `build/`)

## Self-Iteration Workflow

This project is configured to iterate on itself. The workflow:

1. Edit source in `apps/frontend/` or `apps/backend/`
2. Build to `build/` with `npm run build`
3. Launch from `build/` with `npm run build:start` or `npm run dev`
4. The running Aperant instance can then work on its own source code

## Architecture Quick Reference

### Frontend (Electron + React)
- **Entry**: `apps/frontend/src/main/index.ts` ‚Üí builds to `build/main/index.js`
- **Preload**: `apps/frontend/src/preload/index.ts` ‚Üí builds to `build/preload/index.mjs`
- **Renderer**: `apps/frontend/src/renderer/` ‚Üí builds to `build/renderer/`
- **Config**: `apps/frontend/electron.vite.config.ts`
- **State**: Zustand stores in `apps/frontend/src/renderer/stores/`
- **IPC**: Handlers in `apps/frontend/src/main/ipc-handlers/`

### Backend (Python)
- **Entry**: `apps/backend/run.py`, `apps/backend/spec_runner.py`
- **Agents**: `apps/backend/agents/` (planner, coder, session)
- **QA**: `apps/backend/qa/` (reviewer, fixer, loop)
- **Core**: `apps/backend/core/` (client, auth, worktree, platform)
- **Prompts**: `apps/backend/prompts/*.md`

## Coding Standards

### TypeScript/React (Frontend)
- Strict TypeScript ‚Äî `noEmit` checking via `npm run typecheck`
- Biome for linting and formatting ‚Äî `npm run lint` / `npm run lint:fix`
- All UI text via `react-i18next` ‚Äî never hardcode strings
- State management with Zustand 5 ‚Äî stores in `src/renderer/stores/`
- Styling with Tailwind CSS v4 + Radix UI + CVA
- Path aliases: `@/` ‚Üí renderer, `@shared/` ‚Üí shared, `@features/`, `@components/`, `@hooks/`, `@lib/`

### Python (Backend)
- Ruff for linting
- pytest for testing ‚Äî `npm run test:backend`
- Always use `create_client()` from `core.client`, never raw `anthropic.Anthropic()`
- Platform-agnostic via `core/platform/`

### Cross-Cutting
- Never use `process.platform` directly ‚Äî use platform abstraction modules
- Never hardcode file paths ‚Äî use `findExecutable()` and `joinPaths()`
- Dev vs production paths differ ‚Äî always check `app.isPackaged` for Electron paths
- PR target: always `develop` branch, not `main`

- `node_modules/` ‚Äî managed by npm
- `.venv/` ‚Äî managed by pip/uv
- `.auto-claude/` ‚Äî runtime-generated by the app
- `.worktrees/` ‚Äî git worktree isolation
